## 2024-06-11 - Resolving N+1 queries in Spring Data JPA
**Learning:** Spring Data JPA's default behavior for `findAll()` or JPQL queries fetching entities with `@ManyToOne` relationships (like `SanPham.danhMuc`) will trigger N+1 queries if the relationship is lazy, or even if eager when fetched outside of EntityManager's immediate `findById`. The default `findAll` triggers one query for products, and then `N` additional queries for categories as Thymeleaf template accesses `sp.danhMuc`.
**Action:** Use `@EntityGraph(attributePaths = {"<relationship>"})` on the repository methods to instruct Hibernate to perform a `LEFT OUTER JOIN` and fetch the association in a single query. For standard methods like `findAll()`, we can just override it in the repository interface and annotate it.
